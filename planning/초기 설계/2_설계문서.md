# LexiRecall Claude Code 구현 설계문서

이 문서는 Claude Code에 그대로 전달해 공모전용 MVP를 바이브코딩하기 위한 실행 지시서다. 목표는 완성형 상용 서비스가 아니라, 공모전 시연에서 핵심 아이디어가 명확히 보이는 웹앱 MVP를 빠르게 구현하는 것이다.

## 0. Claude Code 첫 요청문

아래 프롬프트를 Claude Code 첫 메시지로 사용한다.

```text
이 프로젝트는 "LexiRecall: 자기주도 법학 백지복습 AI" 공모전 MVP입니다.

목표:
법학 강의자료를 기반으로 개념 지도를 만들고, 사용자가 자신이 안다고 생각하는 개념을 체크한 뒤, 단계별 백지복습을 작성하면 AI가 정답을 직접 알려주지 않고 누락·오개념·이해 검증 질문만 제공하는 웹앱을 구현하세요.

가장 중요한 원칙:
AI는 정답을 직접 설명하는 튜터가 아닙니다.
AI는 학생이 스스로 공부할 수 있도록 최소 단서만 제공하는 점검자입니다.

먼저 다음 문서를 생성하세요.
- CLAUDE.md
- docs/PRODUCT.md
- docs/MVP.md
- docs/USER_FLOW.md
- docs/UI_SPEC.md
- docs/AI_RULES.md
- docs/DATA_MODEL.md
- docs/TASKS.md

그다음 Next.js + Tailwind 기반 MVP를 구현하세요.
처음에는 mock 데이터 기반으로 전체 시연 흐름을 완성하고, 이후 실제 PDF 업로드/AI API/Supabase 연동은 교체 가능한 구조로 분리하세요.

필수 화면:
1. 홈
2. 자료 업로드
3. 개념 지도
4. 백지복습
5. AI 피드백
6. 지식 지도/메타인지 오차

필수 기능:
- 법학 자료 업로드 UI
- 민법총칙 샘플 자료 기반 개념 지도 표시
- 사용자가 알고 있다고 생각하는 개념 체크
- 백지복습 단계 선택
- 백지복습 답안 작성
- 누락·오개념·이해 검증 질문 생성
- 개념별 이해도 색상 표시
- 사전 자기평가와 실제 이해도 차이를 통한 메타인지 오차율 표시

시연 우선순위:
1. 전체 사용 흐름이 끊기지 않아야 합니다.
2. 민법총칙 예시 데이터가 설득력 있어야 합니다.
3. AI 피드백은 정답 해설이 아니라 최소 단서와 질문이어야 합니다.
4. UI는 공모전 심사자가 2분 안에 의도를 이해할 수 있어야 합니다.
5. 실제 백엔드가 없어도 mock 데이터로 데모가 가능해야 합니다.

구현 후 실행 방법과 남은 작업을 README.md에 정리하세요.
```

## 1. 제품 개요

### 1.1 서비스명

LexiRecall: 자기주도 법학 백지복습 AI

### 1.2 한 줄 설명

자기주도학습과 메타인지를 통해 법학 개념의 진짜 이해에 도달하도록 돕는 백지복습 AI

### 1.3 핵심 문제

법학 학습자는 강의자료를 읽고 이해했다고 느끼지만, 막상 백지 상태에서 요건, 효과, 예외, 판례, 비교 개념을 설명하려 하면 누락과 혼동이 발생한다. 기존 AI 튜터는 정답과 설명을 너무 쉽게 제공해 학생의 인출 연습과 자기 점검을 약화시킬 수 있다.

### 1.4 해결 방식

학생이 직접 설명하게 만들고, AI는 학생 답안에서 빠진 부분과 혼동한 부분만 최소 단서로 알려준다. 사전 자기평가와 실제 백지복습 결과를 비교해 메타인지 오차를 시각화한다.

## 2. 최우선 구현 원칙

1. mock 데이터만으로도 처음부터 끝까지 시연 가능해야 한다.
2. 업로드, AI, DB는 실제 연동 전이라도 교체 가능한 함수와 타입으로 분리한다.
3. AI 피드백은 정답 해설이 아니라 누락, 혼동, 검증 질문 중심이어야 한다.
4. 공모전 MVP이므로 로그인, 결제, 다중 사용자, 고급 통계는 구현하지 않는다.
5. UI 첫 화면은 마케팅 랜딩이 아니라 바로 학습 앱이어야 한다.
6. 민법총칙 예시는 구체적이어야 한다. 통정허위표시, 비진의표시, 착오, 무효와 취소, 선의의 제3자를 포함한다.
7. 사용자는 6단계 흐름을 끊김 없이 이동할 수 있어야 한다.

## 3. 권장 기술 스택

### 3.1 MVP 기본

- Framework: Next.js App Router
- Language: TypeScript
- Styling: Tailwind CSS
- UI: shadcn/ui 또는 직접 구현한 Tailwind 컴포넌트
- Icons: lucide-react
- State: React state, 필요 시 Zustand
- Graph: React Flow 권장. 시간이 부족하면 CSS 기반 트리/노드 UI로 대체 가능
- Mock data: `src/lib/mock-data.ts`

### 3.2 실제 기능 교체 시

- DB: Supabase PostgreSQL
- File Storage: Supabase Storage
- PDF parsing: `pdf-parse` 또는 서버 측 PDF parser
- AI: Claude API 또는 OpenAI API
- Deploy: Vercel

### 3.3 구현 판단

시간이 부족하면 Supabase, 실제 PDF 텍스트 추출, 실제 AI API 호출은 하지 않는다. 대신 타입과 함수 인터페이스를 만들어두고 mock 구현을 연결한다.

```ts
export type ConceptType =
  | "unit"
  | "main_concept"
  | "statute"

  | "exception"

  | "complete"

export interface SelfAssessment {

export interface RecallSession {

}
```

```text
- 비진의표시와 통정허위표시는 상대방의 인식 또는 합의 측면에서 어떻게 다를까요?

[다음 복습 단서]
- 민법 제108조 1항과 2항을 나누어 다시 백지복습하세요.
```

## 9. 화면별 구현 상세

### 9.1 홈 `/`

목적:
사용자가 현재 학습 상태를 보고 바로 시연 흐름에 진입한다.

구성:

- 상단 앱 이름: LexiRecall
- 진행 단계 표시: 업로드 → 개념 지도 → 백지복습 → 피드백 → 지식 지도
- 최근 자료: 민법총칙 4주차, 민법총칙 5주차
- 지식 지도 요약: 완성률, 메타인지 오차율, 취약 개념 수
- 주요 버튼: `새 자료 업로드`, `시연 시작`

동작:

- `시연 시작` 클릭 시 `/upload`로 이동

### 9.2 자료 업로드 `/upload`

목적:
파일 업로드 경험을 보여준다. 실제 파일 처리가 없더라도 분석 완료 흐름을 구현한다.

구성:

- 드래그앤드롭 업로드 영역
- 자료명 입력
- 과목명 입력
- 주차 입력
- 분석 시작 버튼
- 분석 진행 상태

동작:

- 파일 선택 또는 샘플 자료 사용 가능
- 분석 시작 클릭 시 1~2초 로딩 후 분석 완료 상태 표시
- 완료 후 `/concept-map` 이동 버튼 표시

### 9.3 개념 지도 `/concept-map`

목적:
AI가 자료에서 뽑은 법학 개념 구조를 보여주고, 사용자가 자기평가를 체크한다.

구성:

- 좌측 또는 상단: 개념 트리
- 각 개념 노드: 타입 배지, 짧은 설명, 체크박스
- 우측 또는 하단: 선택한 개념 상세
- 사전 자기평가 요약: 알고 있다고 체크한 개념 수
- 버튼: `백지복습 시작`

동작:

- 체크박스로 predictedKnown 상태 변경
- 기본적으로 통정허위표시, 비진의표시, 무효와 취소를 알고 있다고 체크된 상태로 둔다.
- `백지복습 시작` 클릭 시 `/recall`로 이동

### 9.4 백지복습 `/recall`

목적:
사용자가 개념을 직접 인출해 작성한다.

구성:

- 복습 대상: 통정허위표시
- 단계 선택 segmented control:
  - 1단계: 목차와 핵심 키워드 제공
  - 2단계: 목차만 제공
  - 3단계: 완전 백지
- 단서 영역
- 답안 작성 textarea
- 예시 답안 넣기 버튼
- 제출 버튼

동작:

- 단계에 따라 단서 표시 내용 변경
- 예시 답안 버튼 클릭 시 mock 학생 답안 삽입
- 제출 클릭 시 `/feedback`으로 이동

### 9.5 AI 피드백 `/feedback`

목적:
정답을 알려주는 것이 아니라, 누락과 혼동을 최소 단서로 알려준다.

구성:

- 학생 답안 요약
- 누락 섹션
- 혼동 섹션
- 이해 검증 질문 섹션
- 다음 복습 단서
- 버튼: `지식 지도 보기`, `다시 복습`

주의:

- 여기서 정답 해설문을 길게 보여주면 안 된다.
- 각 항목은 짧고 구체적이어야 한다.

### 9.6 지식 지도 `/knowledge-map`

목적:
사전 자기평가와 실제 이해도의 차이를 시각화한다.

구성:

- 전체 완성률
- 메타인지 오차율
- 과대평가 개념 목록
- 개념별 색상 지도
- 회차별 변화 mock 그래프 또는 리스트
- 버튼: `취약 개념 다시 복습`

이해도 색상:

- complete: green
- partial: yellow
- misconception: orange
- missing: red

메타인지 오차율 계산:

```text
메타인지 오차율 = 알고 있다고 체크했지만 complete가 아닌 개념 수 / 알고 있다고 체크한 개념 수 * 100
```

## 11. Scoring 로직

`src/lib/scoring.ts`에 구현한다.

필수 함수:

```ts
export function calculateCompletionRate(
  levels: Record<string, UnderstandingLevel>
): number;

export function calculateMetacognitionGapRate(
  assessments: SelfAssessment[],
  levels: Record<string, UnderstandingLevel>
): number;

export function getOverestimatedConceptIds(
  assessments: SelfAssessment[],
  levels: Record<string, UnderstandingLevel>
): string[];
```

규칙:

- completionRate는 complete 개념 수 / 전체 평가 개념 수
- metacognitionGapRate는 predictedKnown이 true인데 complete가 아닌 개념 수 / predictedKnown true 개념 수
- 분모가 0이면 0 반환

## 13. 디자인 요구사항

전체 느낌:

- 법학 공부 도구답게 차분하고 신뢰감 있는 인터페이스
- 정보 밀도가 적당히 높고, 공모전 시연에서 핵심 흐름이 잘 보여야 함
- "AI가 대신 공부해주는 서비스"가 아니라 "학생이 직접 공부하게 만드는 서비스"라는 메시지가 UI에 드러나야 함

구체 요구:

- 상단에 단계 진행 표시를 항상 보여준다.
- 주요 버튼에는 아이콘을 사용한다.
- 개념 노드에는 타입별 배지를 붙인다.
- 이해도 색상 범례를 제공한다.
- 모바일에서는 화면이 세로로 자연스럽게 쌓인다.
- 텍스트가 버튼이나 카드 밖으로 넘치면 안 된다.

금지:

- 과한 랜딩페이지 hero
- 장식용 그라데이션 배경
- 중첩 카드
- 기능 설명만 있고 실제 사용 흐름이 없는 화면

## 14. 공모전 시연 시나리오

2~3분 시연 기준:

1. 홈에서 LexiRecall의 핵심 지표 확인
2. 민법총칙 자료 업로드
3. AI가 통정허위표시 중심 개념 지도를 생성하는 장면
4. 사용자가 "알고 있다고 생각함" 체크
5. 2단계 백지복습 선택
6. 학생 답안 작성 또는 예시 답안 삽입
7. AI가 누락·혼동·검증 질문만 제공
8. 지식 지도에서 통정허위표시 관련 개념이 partial/misconception으로 표시
9. 메타인지 오차율을 통해 "안다고 생각했지만 실제로는 부족한 부분" 확인

시연에서 강조할 문장:

```text
이 서비스는 AI가 정답을 대신 말해주는 것이 아니라, 학생이 스스로 인출하고 자기 오류를 발견하도록 돕습니다.
```

## 20. 가장 중요한 한 줄

LexiRecall은 AI가 답을 알려주는 서비스가 아니라, 학생이 스스로 모르는 것을 발견하게 만드는 법학 백지복습 서비스다.
