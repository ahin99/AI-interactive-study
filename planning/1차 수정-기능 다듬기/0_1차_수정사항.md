
# 이 아래부터는 기능 중심의 자세한 수정사항 


1. 데이터 구조
과목
 └─ 주차
     └─ 업로드 파일
         └─ 개념 -> 주차별 개념 지도
- 한 주차에 파일을 여러 개 올릴 수 있다.
- 한 주차에 올라간 파일은 파일이 여러 개여도 하나의 개념 지도로 묶인다. 즉 백지복습의 단위는 '한 파일' 이 아니라 '한 주차' 다. 주차 중심이다.
- 사용자가 주차 이름은 작성하지 않고, 몇 주차인지만 숫자만 넣어준다.
- 사용자가 파일을 올릴 때 파일 명은 작성한다. (예 : 권리능력)

2. AI API를 붙이기 전 구현 범위
아래 부분에서 최종적으로는 AI가 필요하다.
- 업로드 자료에서 개념 추출
- 개념 지도 생성
- 백지 답안의 누락·오개념 분석
- 소크라테스식 이해 검증 질문 생성
- 학생 답변에 대한 피드백
- 메타인지 정확도 계산

지금은 AI를 연결하지 말고, 모든 결과를 고정된 시연용 데이터로 보여준다. 지금 사용하고 있는 시연용 데이터를 사용하면 된다. 부족한 시연 데이터가 있으면 materials 폴더 내 파일을 참고해 임의로 생성해라.
AI의 구체적 프롬프트는 지금 단계에선 고려하지 않는다.

3. 메인 대시보드의 정확한 구성
- 과목 카드에 표시할 정보
    - 과목명 (예 : 민법)
    - 전체 개념 지도 완성률 (예 : 62%)
    - 메타인지 오차율 (예 : 92% )
    - 마지막 복습일 (예 : 7일 전 )
    - 마지막으로 추천된 복습 내용 (예 : 마지막 복습 단서 - 의사표시 관련 자료를 다시 찾아 백지복습 하세요.)
    - 업로드된 자료 수 (예 : 15개)
- 과목 추가 버튼
- 최근 복습한 과목
- 카드형 UI로 구성. 내가 폴더 느낌이라고 말한 건 실제 폴더 모양 디자인을 뜻하는게 아니라, 단순히 과목별로 묶어 보여준다는 뜻.

4. 과목 상세 화면의 정보 구조
과목 카드를 클릭해 들어가면 과목 상세 화면의 구조 : 
- 지식 지도가 메인에 떠 있다. 
- 메인 화면 지식 지도에서 그 주차 칸을 클릭하면 주차 칸이 확대된다. 확대된 화면 구조 :
    - 해당 주차에 백지복습을 몇 번이나 진행했는지 횟수 (몇회차)
    - 백지복습 하기
        - 백지복습 구조 :
        - 백지복습 전 자기평가 (현재 개념 지도에 해당하는 부분. 체크하는 부분.)
        - 실제 백지복습하기. 단계 선택은 다음과 같음 : 주차의 목차 완전히 다 보여주기 - 주차의 핵심 키워드 3개만 보여주기 - 아예 백지
        - 피드백
        - 지식 지도, 메타인지 지도 갱신
    - 해당 주차의 이전 지식 지도 보기 (성장을 비교하는 용도.) 
    - 해당 주차의 이전 백지복습 내용 보기 (이전에 피드백 받았던 내용을 볼 수 있다.) 
- 지식 지도 확장하기 버튼을 누르면 자료 업로드 가능, 지식 지도가 확장된다.

- 지식 지도, 메타인지 지도는 같은 모양에 색만 다른거다.
- 메타인지 지도 로 전환하기를 누르면 메타인지 지도를 볼 수 있다. 메타인지 지도 역시 지식 지도와 동일하게 그 주차 칸을 클릭하면 칸 확대, 이전 메타인지 지도 보기 가능하다.
- 전체 지식 지도, 메타인지 지도도 이전 - 이후를 볼 수 있다. (변화 과정)

- 지도의 색 변화 과정을 보는 것은, 백지복습 회차별 지도를 저장하여 슬라이더를 움직여 타입랩스처럼 변화를 과거에서 현재까지 볼 수 있는 것을 말한다.
    - 현재 단계에서는 실제 변경 이력을 자동 생성하지 않는다.
    미리 정의된 3~5개의 snapshot 데이터를 사용한다.

    - 슬라이더는 snapshot 인덱스를 이동하며,
    선택한 복습 회차의 지식 지도 또는 메타인지 지도를 표시한다.
    복잡한 애니메이션이나 자동 재생 기능은 이번 범위에서 제외한다.



5. 지식 지도의 구체적 모양
- 지식 지도는 업로드한 자료를 바탕으로 생성된 모자이크형 타일형 개념 지도.
- 개념 지도는 주차 단위로 생성되고, 주차별로 생성된 지도가 묶여 전체 지도를 다시 생성함. 
- 메인 화면에서는 모든 주차의 개념 지도를 한눈에 볼 수 있게.
- 큰 타일 하나가 주차다. (개념이 아니라 주차 기준)
- 전체 메인 화면에서는 주차 단위로 보여주는데, 주차 아래의 개념들을 글자 쓰지 말고 그냥 색깔만 있는 칸으로 나태내서 보여준다. 즉 사용자가 색이 모자이크 칸처럼 어떤 수준인지 볼수 있게.
- 주차를 클릭하면 그 안의 개념 조각 타일이 글자까지 포함해서 확대되어 나타난다.
    - 주차 클릭 -> 조각 타일 확대 이 상황에서 해당 주차 백지복습 하기 버튼이 나온다. 이걸 누르면 백지복습 시작.
    - 확대는 줌인 애니메이션으로.
- 조각의 크기는 모두 동일.


6. 지식 지도 색상의 의미와 단계
지식 지도 색상 설명. 
- 답변이 전혀 없거나 핵심 내용을 전혀 언급하지 않음 → 빨강
- 내용을 작성했으나 오개념이 존재함 → 주황
- 오개념은 없지만 핵심 요소 일부가 빠짐 → 노랑
- 핵심 내용을 정확히 설명함(완전히 이해한 상태) → 초록
- 평가 기록 없음 → 회색
- 보라 -> 백지복습 상태에서는 정확히 설명했지만 이해 검증 질문에서 틀렸음. 즉 내용은 기억하지만 심층 이해가 부족함.

메타인지 위험도 지도
- 짙은 핑크 - 옅은 핑크로 갈수록 : 메타인지 오차율이 높음 - 메타인지 오차율이 낮음
- 메타인지 오차율은 학생이 알고 있다고 체크한 개념 중 지식 지도 색상의 결과를 뺀 수치다. 즉 메타인지 오차율이 낮을수록 학생이 메타인지가 잘되고 있는거니까 좋은것.
- 메타인지 지도는 과대평가만 중요하게 본다. (과소평가는 고려 안함)
- 정확한 건 나중에 AI를 연동할 때 공식을 만들 예정이니 시연용 데이터에서는 정확한 공식 고려하지 말것.

    현재 단계에서는 정확한 계산식을 구현하지 않는다.

    시연 데이터에서는 각 개념마다 metacognitionGap 값을
    0~100 사이의 고정 숫자로 미리 저장한다.

    값이 클수록 과대평가가 큰 것으로 간주하며,
    지도 색상은 해당 값에 따라 옅은 핑크에서 짙은 핑크로 표시한다.


7. 첨삭형 피드백 UI

- 전체 화면 구성 개요 :
    - 학생이 백지복습한 글에 대한 첨삭
    - 이해 검증 질문
    - 추천 복습 내용

통정허위표시는 당사자 사이에서는 취소할 수 있다.
──────────────────────────────
[오개념] 설명~~~

위와 같은 형태로 구현. 

- 학생 답안은 사용자가 입력한 줄바꿈을 기준으로 블록을 분리한다.
- 각 블록 안에서는 문장 단위로 분석하여 필요한 문장에 형광펜을 표시한다.
- 잘못된 부분에 형광펜 표시를 한다. (밑줄 아님.) 형광펜 표시는 학생의 잘못된 유형에 따라 지식 지도 색상과 동일하게 한다. (예 : 오개념이 있으면 주황 형광펜. 일부 누락이면 노랑 형광펜.) 
- 두 색이 겹치는 경우 형광펜의 우선순위는 주황 > 노랑

- 누락된 내용은 학생 글에 대응 문장이 없으니, 학생 글이 다 끝나고 나서 그 아래 빨간 글씨로 표시한다.
- 주의점 : 피드백할 때, 절대 정확한 답을 알려주지 않는다. 예를 들어 '통정허위표시는 취소할 수 있는 게 아니라 무효입니다.' 하고 알려주지 않는다. 왜냐하면 이 프로그램은 정답을 알려주는 AI가 아니라 단서만 줘서 학생이 직접 생각해 볼 수 있게 하는 AI이기 때문이다.
    - 허용하는 피드백 수준:
        - 대원칙 : 절대 정답을 직접 말하지 말것.
        - 다시 확인해야 할 법적 쟁점을 알려줌
        - 서로 구분해야 하는 기준을 질문 형태로 제시함
        - 관련 개념명 정도는 제시 가능
        - 모범답안이나 정정 문장 제공 금지
            

- 올바르게 설명한 부분에는 초록색 형광펜 표시를 한다. 피드백은 작성하지 않는다.

- 올바르게 설명한 부분의 내용에는 소크라테스식 이해 검증 질문을 3개 해 준다. (맞게 썼지만 진짜로 이해하고 있는지 알아보기)
- 이해 검증 질문은 줄바꿈 밑에 달아주는 게 아니라, 따로 밑에 별도의 칸에 해준다. 
- 질문별로 학생이 답을 타이핑해서 입력한다. 
    - 이해 검증 질문의 유형:
        - 이유 질문
        - 경계 질문
        - 비교 질문
        - 적용 질문
        - 전제 질문
- 이해 검증 질문을 포함한 피드백에 대한 '정답' 은 제공하지 않음. 
- 이해 검증 질문도 '질문만' 던짐. (답을 찾는 공부는 학생의 몫)
- 이해 검증 질문에 대한 답을 틀렸으면 백지 복습에서 올바르게 썼어도 지도에서 초록색이 아닌 보라색으로 들어간다. 틀렸는지 아닌지는 학생에게 알려주지 않고 AI가 지식 지도 재구성할때 반영한다.

- 마지막 피드백 총평으로, 학생이 다음 복습하면 좋은 내용을 한 문장으로 추천해 준다.

8. 개념 지도 완성률 이란, 전체 타일에서 초록색(완전 이해) 칸의 개수를 말한다.

9. 전체적인 디자인 느낌 지시
- 전문적인 느낌
- 웹이므로, 모바일보다는 데스크톱 우선으로 디자인할것. 
- 흰색 또는 매우 옅은 회색 배경
- 카드 사이 간격을 충분히 둠

- 한 화면에 지나치게 많은 텍스트를 표시하지 않음
- 핵심 지표는 큰 숫자로 표시
- 지도는 가장 큰 시각적 영역을 차지함
- 버튼과 카드 모서리는 중간 정도로 둥글게
- 장식보다 정보의 위계와 가독성을 우선함
- 컬러는 지도 상태 표현에 집중하고, 일반 UI에는 과도하게 사용하지 않음
