# 요구사항

현재 프로젝트 폴더에는 다음이 있습니다.

* Claude Code가 1차로 구현한 웹앱 소스 코드: lexirecall 폴더 안 코드
* 사용자가 작성한 최종 수정 요구사항 문서: `2차_수정사항.md`
* 시연용 학습자료가 들어 있는 `materials` 폴더

이 프로젝트는 자기주도학습과 메타인지를 통해 진짜 이해를 돕는 법학과용 백지복습 AI 학습 도우미 웹앱입니다.

이번 작업에서는 코드를 수정하지 말고, 현재 프로젝트 코드와 `2차_수정사항.md`를 분석하여 Claude Code CLI가 바로 구현 작업에 사용할 수 있는 최종 수정 설계 문서를 planning 폴더 안에 작성해 주세요.

생성할 파일명:

`최종_수정사항설계문서.md`

## 작업 순서

1. 프로젝트의 디렉터리 구조와 기술 스택을 확인하세요.
2. 주요 페이지, 라우트, 컴포넌트, 타입, 상태 관리, mock 데이터 위치를 확인하세요.
3. 현재 사용자 흐름과 구현된 기능을 분석하세요.
4. `2차_수정사항.md`를 읽고 현재 코드와 목표 상태의 차이를 정리하세요.
5. 그 결과를 바탕으로 실제 구현 가능한 수정 설계 문서를 작성하세요.

## 중요한 제한

* 애플리케이션 코드를 수정하지 마세요.
* 기존 파일을 삭제하거나 이동하지 마세요.
* 패키지를 설치하지 마세요.
* 실제 AI API를 연결하지 마세요.
* AI 프롬프트를 설계하지 마세요.
* `최종_수정사항설계문서.md`만 생성하세요.
* 존재하지 않는 파일을 기존 파일인 것처럼 적지 마세요.
* 가능한 한 현재 프로젝트의 실제 파일 경로를 사용하세요.

## 요구사항 해석 원칙

`2차_수정사항.md`를 최우선 기준으로 사용하세요.

다음 원칙은 반드시 유지하세요.

* 학습 단위는 파일이 아니라 주차
* 한 주차에 여러 파일 업로드 가능
* 같은 주차의 파일은 하나의 개념 지도로 통합
* 백지복습이 서비스의 핵심
* AI는 정답을 직접 제공하지 않고 생각할 단서만 제공
* 현재 단계에서는 기존 시연 데이터와 mock 데이터를 사용
* 지식 지도와 메타인지 지도는 같은 타일 구조를 사용
* 실제 AI API와 정교한 메타인지 계산 공식은 이번 범위에서 제외

문서 안에 충돌하거나 불명확한 부분이 있으면 합리적인 기본안을 제안하되, 확정된 요구사항처럼 숨기지 말고 `구현 전 확인 사항`에 따로 적어 주세요.

## 반드시 구체적으로 작성할 내용

각 기능에 대해 가능하면 다음을 적어 주세요.

* 현재 관련 파일
* 수정 또는 신규 생성 여부
* 화면에 보이는 요소
* 사용자 행동
* 저장하거나 변경되는 데이터
* 사용할 컴포넌트
* 빈 상태와 오류 상태
* 구현 완료 확인 방법

## 현재 요구사항에서 명확히 정리할 규칙

다음 기준을 사용하세요.

### 지식 지도 상태

* 회색: 미복습
* 빨강: 핵심 내용을 전혀 작성하지 못함
* 주황: 오개념 포함
* 노랑: 일부 핵심 내용 누락
* 초록: 정확히 설명함

백지 답안이 정확하지만 이해 검증 답변이 부족한 경우에는 주황이 아니라 노랑으로 처리하세요.

### 메타인지 지도

* 짙은 핑크: 과대평가 오차가 큼
* 옅은 핑크: 과대평가 오차가 작음
* 회색: 미측정

현재는 개념별로 0~100의 고정 mock 값을 사용하고, 정확한 계산식은 구현하지 마세요.

### 학습 흐름

과목 선택
→ 주차 선택
→ 사전 자기평가
→ 백지복습 난이도 선택
→ 백지 답안 작성
→ 첨삭형 피드백
→ 이해 검증 질문 3개 답변
→ 정답 공개 없이 결과 저장
→ 지식 지도와 메타인지 지도 갱신

### 피드백 원칙

정답 문장과 모범답안을 직접 제공하지 마세요.

대신 다음만 제공합니다.

* 다시 확인할 쟁점
* 구분해야 할 기준
* 질문 형태의 단서
* 부족한 사고 지점

## 최종 검토

문서를 완성한 후 다음을 스스로 확인하세요.

* 현재 코드 분석이 반영되었는가
* `2차_수정사항.md`의 요구가 빠지지 않았는가
* 사용자 흐름이 처음부터 끝까지 연결되는가
* 실제 파일 경로가 반영되었는가
* 코드 수정 없이 문서만 생성했는가
* Claude Code가 문서만 보고 바로 구현을 시작할 수 있는가

문서는 한국어 Markdown으로 작성하세요.
